# Vantage: Protocols as Verifiable Experiments

## Auditor Lens (Self-Correction)
*   **The Sunk Cost Trap:** Are we keeping this protocol because it is the best expression, or because we've invested engineering cycles?
*   **The Falsifiability Test:** Can we state exactly what evidence would require us to abandon this protocol?
*   **The Dead Weight Audit:** If this protocol were deleted, what functionality *actually* vanishes, and is that functionality worth the maintenance of this protocol?

## Practical Application
*   **Protocol as Hypothesis:** Every ADR is an experiment.
*   **Protocol as Discardable Asset:** If the test fails, we branch/fork the state, not patch the code.
*   **Protocol as History:** The failed experiment remains in the Vault, but is evicted from the Manifest.
