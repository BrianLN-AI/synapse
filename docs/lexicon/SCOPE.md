# SCOPE.md — The Boundary of Visibility

**Status:** Soft Definition (f_0-f_11 baseline)
**Updated:** Purged "Vault" in favor of "World"

---

## The Soft Definition

**Scope** is the answer to the question: **"What can I reach from here?"** 

It is the **Closure** of reachable expressions in the **World**. If I have a reference (a hash) to Expression A, and A contains a reference to B, then both A and B are in my scope.

---

## Shapes

### [ALGEBRAIST] — The Closure
Scope is the **transitive closure** of a set of starting hashes in the **World Graph**. It is a **Relational** property.

### [PHYSICIST] — The Light Cone
Scope is the **Observable Universe** of a specific execution. A blob can only "see" and "interact with" objects that are causally connected to it via hash-references in the **World**. Anything outside the scope is physically invisible to the Isolate.

### [JURIST] — The Jurisdiction
Scope defines the **Territory** where a set of rules (verbs) applies. If an expression's hash is in your scope, you have the "standing" to attempt a **Message** (INVOKE). 

---

## Comparison

- **Scope vs. Partition:** Scope is about **Reachability** (Graph-based); Partition (Tenancy) is about **Residency** (Set-based / Boundary-based).
- **Scope vs. Ownership:** Scope is about **Visibility** (transitive); Ownership is about **Signature** (who authored it).
- **Scope vs. Authorization:** Scope is **"Can I see it?"**; Authorization is **"Am I allowed to use it?"**
