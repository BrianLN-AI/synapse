# f_3: D-JIT Collective Intelligence
**Version:** 1.3.0-Alpha
**Status:** Logical Architecture & Collective Protocol

## 1. Executive Summary
The **D-JIT (Distributed Just-In-Time) Logic Fabric** is a content-addressed, protocol-oriented compute mesh. `f_3` evolves the fabric from an individual agentic system into a **Collective Intelligence**. It bridges independent Synapse instances into a global, peer-to-peer compute substrate.

## 2. Goals & Objectives (GOKR)
* **Goal:** A global, decentralized compute mycelium with shared discovery and federated arbitrage.
* **Objective:** Enable location-transparent execution across independent fabric nodes.
* **Key Result:** A Librarian capable of resolving hashes via a global DHT (Decentralized Hash Table).

## 3. Logical Architecture: The 4-Layer Stack
1. **Interface (The Proxy):** Captures intent and translates it into a Protocol Contract. (f_2: MCP-Enabled)
2. **Discovery (The Librarian):** Resolves Content-Hashes across vault tiers. (f_3: P2P Collective resolution added)
3. **Planning (The Broker):** Performs "Market Arbitrage" (Latency vs. Cost vs. Trust). (f_3: Federated Arbitrage across nodes)
4. **Binding (The Engine):** Physically injects the Blob into a scrubbed sandbox. (f_2: Retries & Memory)

## 4. The Execution Contract (The ABI)
To ensure deterministic execution, all Blobs must adhere to the **Scope-Exchange Protocol**:
* **Inputs:** The Linker injects a `context` object (data), a `log(msg)` function (telemetry sink), and an optional `state` object.
* **Outputs:** Blobs **must** assign their final return value to a `result` variable in the local scope. Raw `return` statements are prohibited.
* **Isolation:** Blobs are executed in a "Scrubbed Scope." They cannot see Linker internal variables (e.g., `VAULT_DIR`) unless explicitly passed.

## 5. The Bootstrap Protocol (The BIOS)
To prevent infinite recursion (where Discovery needs Discovery to find Discovery):
* **The BIOS Fallback:** The Seed Linker MUST implement a private `_raw_get(h)` method.
* **The Root Pointer:** This method bypasses Blob-based Discovery to resolve the initial System Manifest directly from the filesystem.

## 6. Seed Functional Requirements
Implement a **Seed CLI/MCP Server** in Python (`seed.py`) and JavaScript (`seed.js`):
1. **PUT**: Accept Type/Payload -> Hash -> Store in `./blob_vault/`.
2. **INVOKE**: Load Hash -> Bind (exec/eval) -> **Verify `result` exists** -> Return.
3. **TELEMETRY**: Measure latency/memory and `PUT` a 'telemetry/artifact' blob for every run, including any `log()` data.
4. **STATE**: Maintain a `blob_vault/state/` directory for persistent `state_id` mapping.
5. **COLLECTIVE**: Maintain a `collective_registry` for cross-node hash discovery.
